# Azure Network Security Groups (NSGs) and Traffic Analysis Between Azure Virtual Machines

## Project Overview

This project demonstrates how to deploy and configure Azure networking resources while analyzing network traffic between Windows and Linux virtual machines using Wireshark. Throughout the lab, I explored how common networking protocols function and how Azure Network Security Groups (NSGs) can be used to allow or deny network communication.

The project provided hands-on experience with Azure networking, virtual machine administration, firewall configuration, packet analysis, and troubleshooting network connectivity.

## Lab Activities

### 1. Azure Infrastructure

* Created a Resource Group.
* Created a Virtual Network and Subnet.
* Deployed a Windows 10 Virtual Machine.
* Deployed an Ubuntu Linux Virtual Machine.
* Connected both VMs to the same Virtual Network.

---

### 2. ICMP Packet Analysis

* Installed Wireshark.
* Started packet capture.
* Filtered ICMP traffic.
* Successfully pinged the Ubuntu VM.
* Observed Echo Requests and Echo Replies.

---

### 3. Network Security Group (Firewall)

* Configured inbound firewall rules.
* Blocked ICMP traffic.
* Verified ping requests failed.
* Re-enabled ICMP traffic.
* Confirmed connectivity was restored.

---

### 4. SSH Traffic

* Connected to the Ubuntu VM using SSH.
* Executed Linux commands.
* Captured encrypted SSH traffic with Wireshark.

---

### 5. DHCP Traffic

* Renewed the Windows VM IP address.
* Observed DHCP traffic generated by the renewal.

---

### 6. DNS Traffic

* Used `nslookup` to resolve domain names.
* Captured DNS queries and responses.

---

### 7. Remote Desktop Protocol (RDP)

* Filtered traffic for TCP port 3389.
* Observed continuous RDP communication between the client and the Windows VM.
